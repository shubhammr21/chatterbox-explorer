# Implementation Checklist

## Pre-Coding Validation
- [ ] PLAN.md is followed strictly
- [ ] No new assumptions introduced
- [ ] Tools/libraries decision confirmed

## Coding Principles
- [ ] Each function has a single responsibility
- [ ] Code is readable (not clever)
- [ ] Naming is clear and meaningful
- [ ] No unnecessary abstraction

## Tooling & Ecosystem
- [ ] Using existing library/framework where appropriate
- [ ] Following official docs/examples (not generic code)
- [ ] Design pattern adhered to

## Validation
- [ ] Logic verified with sample inputs/outputs
- [ ] Edge cases handled

## Safety
- [ ] No hidden hacks/workarounds
- [ ] Limitations clearly visible in code/comments

## Sync Back
- [ ] Any new findings added to NOTES.md
- [ ] Any design changes reflected in PLAN.md

❗ If deviation from plan → STOP and update PLAN.md first
