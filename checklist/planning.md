# Planning Checklist

## Understanding
- [ ] Problem is clearly defined in 2–3 lines
- [ ] Requirements are complete and unambiguous
- [ ] Unknowns are identified
- [ ] No assumptions remain (or explicitly listed)

## Research & Knowledge
- [ ] Relevant docs/examples reviewed
- [ ] Existing solutions/libraries explored
- [ ] Notes added to NOTES.md (if new learning)

## Design
- [ ] PLAN.md created/updated
- [ ] Inputs / Outputs defined
- [ ] Constraints identified
- [ ] Edge cases listed
- [ ] Risks identified

## Architecture
- [ ] Approach is broken into smaller units
- [ ] Flow/logic diagram created (Mermaid if needed)
- [ ] Design pattern decided (if applicable)

## Decision Clarity
- [ ] Alternatives considered
- [ ] Chosen approach justified
- [ ] DECISIONS.md updated (if needed)

## Readiness Gate
- [ ] Plan is reviewed
- [ ] No critical doubts remain

❗ If ANY item is incomplete → STOP and resolve before coding
