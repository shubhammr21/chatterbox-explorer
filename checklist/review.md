# Review & Improvement Checklist

## Code Quality
- [ ] Code is simple and readable
- [ ] No duplication
- [ ] Functions/modules are well-scoped

## Design Integrity
- [ ] Matches original design (PLAN.md)
- [ ] No unintended complexity added
- [ ] Design patterns correctly followed

## Testing & Validation
- [ ] Critical paths tested
- [ ] Edge cases verified
- [ ] Failure scenarios handled

## Improvement Opportunities
- [ ] Can this be simplified further?
- [ ] Can performance be improved?
- [ ] Can readability be improved?

## Knowledge Capture
- [ ] Learnings added to NOTES.md
- [ ] Decisions documented in DECISIONS.md

## Final Check
- [ ] No assumptions introduced
- [ ] No unclear logic remains

❗ If improvement found → propose BEFORE refactoring
