# REFACTOR FLOW

==ANALYSIS MODE==

## Understanding
- [ ] Existing code fully understood
- [ ] Current behavior documented
- [ ] Tests available (or added)

## Refactor Goal
- [ ] Objective defined (readability / performance / structure)
- [ ] Code smells identified (duplication, complexity)

## Safety Check
- [ ] No feature changes included
- [ ] No bug fixes mixed with refactor

==EXECUTION MODE==

## Refactoring Rules
- [ ] Small incremental changes
- [ ] Maintain same behavior
- [ ] Improve structure step-by-step

## Patterns
- [ ] Apply design patterns only if needed
- [ ] Avoid forced abstraction

## Validation
- [ ] Tests pass after each step
- [ ] No regression introduced

==REVIEW MODE==

## Quality Check
- [ ] Code simpler than before
- [ ] Duplication reduced
- [ ] Naming improved

## Documentation
- [ ] Changes explained in NOTES.md
- [ ] Decision recorded if major

❗ If behavior changes → rollback immediately
